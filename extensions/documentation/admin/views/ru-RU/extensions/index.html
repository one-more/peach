Установщик расширения должен быть реализован в виде исполняемого архива. Напрример: hello_word.phar.php. В архиве должен быть файл install.ini.
Файлом-установщиком должен быть файл install.php. Он должен быть доступен по ссылке "phar://{$phar}/install.php", где $phar - путь к архиву.
Возможные варианты файла install.ini: 
                    [info] 

                    type=extension 

                    name=hello_world 

                    is_daemon = "true"|"false" 

                    daemon_type=admin|site (если is_daemon=true) 

                    cms_version=1.5
                
                    [info] 

                    type=template 

                    name=test 

                    template_type=site|admin 

                    cms_version=1.5 

                
                    [info] 

                    type=editor 

                    name=test 

                
                    [info] 

                    type=update 

                    for=cms|extension_name 

                    version=1.2 

                
                    [info] 

                    type=view 

                    for=extension_name 

                
                    [info] 

                    type=patch 

                
                /admin 

                /admin/controllers/ 

                /admin/models/ 

                /site  

                /site/controllers/ 

                /site/models/ 

                /extension.ini 

                /extension.php
Данные расширения должны реализовывать интерфейс user_extension_interface. Если расширение имеет виджеты, оно должно реализовывать widget_extension_interface.
Такие расширения реализуют определенный функционал по управлению контентом.
Данные расширения должны реализовывать интерфейс menu_extension_interface.
Такие расширения реализуют навигацию на сайте.
Данные расширения должны реализовывать интерфейс daemon_extension_interface
Такие расширения работают "в фоне" - каждый раз, когда в административной панели пользователь переходит по ссылке, загружаются js файлы расширения.
Данные расширения должны реализовывать интерфейс editor_interface.
Такие расширения реализуют функционал визуального редактора
Данные расширения должны реализовывать интерфейс admin_template_interface или site_template_interface.
Такие расширения являются шаблонами для сайта или административной части.
Расширение входит в состав ядра.
system::$system_js
Массив js файлов. Необходимо подключать в административном шаблоне.
system::$system_site_js
Массив js файлов. Необходимо подключать в шаблоне сайта.
system::get_languages()
Возвращает массив доступных в системе языков
system::get_current_language($file = false)
Возвращает текущий язык, установленный в системе. Если передаваемый параметр true, значения кук будут проигнорированы и вернется значение из файла configuration.ini
system::get_menu()
Вернет обработчик меню. Если обработчик не установлен, вернет -1.
Расширение входит в состав ядра. Реализует работу с пользователями.
user::is_auth()
Если пользователь в системе, вернет true, иначе false
user::is_admin()
Если текущий пользователь администратор, вернет true
user::is_super_admin()
Если текущий пользователь супер администратор, вернет true
user::get_token()
Вернет токен сессии, если пользователь в системе и токен верный
