Библиотека классов

builder::build($name, $array, $tag = true)
Собирает массив js или css файлов в один.
Параметры:
$name - имя результирующего файла. Например extension_default.js
$array - массив файлов
$tag - если true, метод вернет тэг , иначе вернет путь
Класс для работы с comet сервером.
comet::add_message($array, $type = 'me_admin')
Параметры:
$array - массив параметров:
task - delegate
object - JavaScript класс(объект). Этот объект должен быть глобальным, т.е. доступным через window[object]
method - метод объекта
params - массив переметров, передаваемых в метод; массив не должен быть ассоциативным
$type - куда передавать сообщение. Возможные варианты:
me_admin - текущему(вошедшему) пользователю в административную часть
me_site - текущему(вошедшему) пользователю на сайте
site_users - всем пользователям на сайте
так же можно передавать ip адрес конкретного пользователя
document::$css_files - Массив css файлов, необходимых CMS
document::$js_files - Массив js файлов, необходимых CMS
Данные файлы необходимо подключать в шаблонах для сайта и административной части
dom::create_element($tag, $params)
Создает html элемент и возвращает его в виде строки
Параметры:
$tag - название тэга, может быть как в фигурных скобках <>, так и без них
$params - массив переметров, таких как id, class и т.п.
factory::getIniServer($path = null)
Возвращает объект класса iniServer. Если путь не задан, целевым файлом будет configuration.ini
factory::get_reference($section, $lang = 'en-EN')
Вернет массив с языковыми переменными из файла /lang/references/$lang.ini
Параметры:
$section - название секции
$lang - язык по умолчанию, если нет файла для текущего языка
Класс содержит общие методы.
helper::remDir($path)
Удаляет директорию.
helper::copy_dir($source, $dest)
Копирует директорию
helper::getSql($path)
Возвращает массив запросов. Запросы в файле должны быть разделены пустой строкой.
Параметры:
$path - путь к sql файлу
helper::purify($array)
Очищает массив элементов от html сущностей
helper::delete_empty_values($array)
Удаляет пустые элементы в массиве
Класс для работы с ini файлами.
$iniServer->read($section, $key, $def = '')
Возвращает значение из файла.
Параметры:
$section - секция
$key - ключ
$def - значение по умолчанию
iniServer->readSection($section, $def = [])
Возвращает ассоциативный массив данных секции.
iniServer->write($section, $key, $value)
Записывает значение в файл.
Параметры:
$section - название секции
$key - ключ
$value - значение
iniServer->writeSection($section, $array)
Записывает массив в секцию
iniServer->writeAll($array)
Записывает массив в ini файл
iniServer->get_all()
Возвращает массив со всем содержимым файла
iniServer->eraseSection($section)
Удаляет секцию
iniServer->deleteKey($section, $key)
Удаляет ключ секции
iniServer->readSections(&$array)
Помещает названия всех секций в массив, переданный в метод
iniServer->readKeys($section, &$array)
Возвращает массив ключей секции
Параметры:
$section - название секции
$array - массив, куда будет помещен результат
iniServer->updateFile()
Данный метод необходимо вызывать для обновления и сохранения данных в файле
От данного класса необходимо наследовать все контроллеры.
supercontroller->exec($task, $params = null)
Данный метод позволяет вызвать метод класса.
Параметры:
$task - название класса
$params - параметры, передаваемые в метод
Вернет значение, которое вернет метод. Если запрос был выполнен методом ajax($_REQUEST['ajax'] == 1) и метод вернет массив, массив будет преобразован в json формат
От данного класса необходимо наследовать все модели.
В классе подключается трейт trait_valid
protected $_db
Поле содержит экземпляр класса PDO.
static $reference
Содержит массив языковых переменных: ['not_empty', 'email', 'positive_number', 'password']
superModel->get_all($table, $start = null, $per_page = null)
Возвращает записи из таблицы. Если параметры $start и $per_page не переданы, вернет все записи.
Параметры:
$table - название таблицы
$start - индекс начальной записи
$per_page - количество записей
superModel->getTotal($table)
Вернет количество записей в таблице
Параметры:
$table - название таблицы
superModel->execute($task)
Выполнит запрос
Параметры:
$task - строка с sql запросом
Шаблонизатор
templator::getTemplate($tpl, $params, $path)
Вернет обработанный html файл.
Параметры:
$tpl - название html файла без расширения
$params - ассоциативный массив с параметрами. В html файле переменные обозначаются с двоеточием - :variable. В данном случае массив с параметрами будет выглядеть так: ['variable' => '123']; Так же допускается конструкция %v {something} v% - если $params[v] не пустое значение, подставиться оно - иначе останется {something}
$path - путь к html файлу, не включая сам файл. Например: SITE_PATH.'extensions'.DS.'test'.DS.'admin'.DS.'views'.DS.'default'
templator::getReplaced($file, $params)
Вернет обработанный файл. В отличии от getTemplate() файл может быть с любым расширением
templator::get_warning($text)
Вернет блок с предупреждением.
Параметры:
$text - текст предупреждения
